# Reflections on Trusting Trust
Ken Thompson

## Introduction 
컴퓨터 시스템의 신뢰는 근본적으로 불가능하며, 누군가를 맹목적으로 믿어야만 한다

## Stage I
self-reproducing program 
자기 자신의 소스 코드를 아웃풋으로 산출하는 프로그램 


## Stage II 
v1 컴파일러 소소의 이스케이프 문자 식별 소스엔 특정 문자(e.g. '\v', vertical tap character)가 하드코딩 되어 있지 않다.

"\v"를 만났을 때 '\v'를 리턴하는 코드를 추가하고 싶다고 하자. 그럼

1. "\v"를 만나면 아스키 코드 11을 리턴하는 코드를 추가하고 자기 자신을 컴파일 한다
2. 11은 이해하기 직관적이지 않으므로 리턴의 11을 '\v'로 고친다
3. 2를 컴파일한다. 그때 '\v'는 11로 치환되어 컴파일된다. 

이 과정을 통해 컴파일러가 '\v'를 이해하는 방식이 '유전된다' 

## Stage III
위 방식을 사용해 컴파일러에 악의적인 코드를 삽입할 수 있다. 

예시로:
1. 컴파일러가 유닉스를 컴파일할 때, 유닉스의 login 함수를 패턴매칭을 통해 인식한다.
2. 유닉스 login 함수가 맞다면, 악성 로직(백도어 1: 사용자가 의도한 비밀번호거나, *공격자가 임의로 추가한 마스터키 비밀번호*면 로그인 성공)을 코드에 추가한다. 
3. 이 컴파일러 프로그램이 자기자신을 컴파일할 때, 백도어 1 로직을 결과물에 추가하는 코드도 추가한다(백도어 2)

백도어 1과 2 코드를 추가한 후 컴파일을 한다.  그리고 해당 컴파일러의 백도어 1, 2 코드를 지우고 다시 한번 컴파일을 한다면 소스코드에 흔적이 지워지는 동시에 백도어 1, 2의 기능은 유전된다. 

이런 식으로 오염된 컴파일러는 다른 방식으로 빌드되었고 깨끗한 컴파일러로 컴파일 하지 않는 이상 악성코드를 탐지하기 어렵다. 

# 파훼법 
## 다양한 이중 컴파일 (Diverse Double-Compiling, DDC)
검증 방법:
1. 의심스러운 컴파일러(A)의 소스코드를 준비한다.
2. 이 소스코드를 오염되었을지도 모르는 컴파일러(A)로 컴파일하여 실행 파일(X)을 만든다.
3. 동일한 소스코드를, 기원과 개발자가 완전히 다른 신뢰할 수 있는 컴파일러(B)(예: 해커가 건드리지 못했을 아주 오래된 다른 언어의 컴파일러 등)로 컴파일하여 실행 파일(Y)을 만든다.
4. X와 Y의 동작(혹은 결과물)을 비교한다.

## 최소 신뢰 기반 부트스트래핑 (Bootstrappable Builds)
'닭과 달걀'의 딜레마를 근본적으로 끊어내기 위해, 달걀을 아주 원시적인 상태부터 인간의 손으로 직접 빚어내는 방식. 
현재 리눅스 생태계(GNU Mes, Stage0 프로젝트 등)에서 실제로 진행 중인 방어책이다.

1. 초기화: 인간이 눈으로 기계어를 100% 읽고 검증할 수 있을 만큼 극도로 단순하고 짧은 헥스(Hex) 코드나 어셈블리어로 아주 작은 미니 컴파일러를 만든다. (이 코드는 너무 짧아서 백도어를 숨길 공간조차 없다.)

2. 단계적 성장: 이 '절대적으로 깨끗한 미니 컴파일러'를 이용해 조금 더 큰 C 컴파일러를 빌드하고, 그 C 컴파일러로 최신 GCC 컴파일러를 빌드한다.

결과: 신뢰의 뿌리가 '인간이 100% 검증한 300줄짜리 기계어'에서 시작되었기 때문에, 중간에 누군가 오염된 컴파일러 바이너리를 끼워 넣을 틈이 원천 차단된다.

## 재현 가능한 빌드 (Reproducible Builds)
현대 소프트웨어 공학에서 가장 중요하게 밀고 있는 보안 표준 중 하나이다.

원리: 똑같은 소스코드를 주면, 전 세계 누가 언제 어느 컴퓨터에서 컴파일하든 토씨 하나 틀리지 않고 100% 완벽하게 똑같은 바이너리(비트 단위까지 일치)가 나오도록 빌드 환경을 엄격하게 통제한다.

방어 효과: 만약 내 컴퓨터에 설치된 컴파일러가 켄 톰슨의 백도어에 감염되어 몰래 악성 코드를 섞어 넣었다면, 내 컴퓨터에서 나온 최종 바이너리의 해시값(Hash)이 전 세계 표준 해시값과 달라지게 된다. 

누구나 독립적으로 컴파일해 보고 결과를 대조할 수 있으므로, 단일 지점에서의 오염 시도를 즉시 적발할 수 있다.
